# Инструкция: как выложить сайт в интернет

## Вариант 1 — GitHub Pages (бесплатно, навсегда)

**Один раз — включить Pages:**

1. Открыть `github.com/andrgavrilenko/medcall`
2. Вкладка **Settings** (наверху страницы)
3. Слева в меню → **Pages**
4. **Branch:** выбрать `main`, папка `/  (root)` → нажать **Save**
5. Подождать ~1 минуту → сайт будет по адресу:

```
https://andrgavrilenko.github.io/medcall/
```

**Обновить сайт после изменений:**

Когда поменял что-то в `index.html` — просто сохранить файл, а потом в терминале:

```bash
cd C:\Users\andrg\coding-projects\Medical
git add index.html
git commit -m "описание изменений"
git push
```

Сайт обновится автоматически через 1–2 минуты.

---

## Вариант 2 — Netlify (быстрее, проще обновлять)

**Первый раз:**

1. Открыть `app.netlify.com` → войти через GitHub
2. Нажать **Add new site → Import an existing project**
3. Выбрать GitHub → найти репозиторий `medcall`
4. Нажать **Deploy** — сайт сразу в интернете

Netlify автоматически обновляет сайт каждый раз, когда делаешь `git push` — ничего больше делать не нужно.

Адрес будет примерно: `https://medcall-abc123.netlify.app/`
Можно поменять на красивый в настройках (бесплатно).

---

## Что выбрать?

| | GitHub Pages | Netlify |
|---|---|---|
| Цена | Бесплатно | Бесплатно |
| Скорость обновления | 1–2 мин | ~30 сек |
| Свой домен | Да | Да |
| Сложность | Минимальная | Минимальная |

Оба варианта подходят. Netlify чуть удобнее в долгосрочной перспективе.

---

## Подключить свой домен (опционально)

Если есть домен (например `medcall.ru`):

**GitHub Pages:** Settings → Pages → Custom domain → вписать домен
**Netlify:** Site settings → Domain management → Add custom domain

У регистратора домена нужно добавить DNS-запись типа `CNAME` или `A` — Netlify/GitHub покажут конкретные значения при настройке.
